8. You MUST implement a real tokenizer + parser + evaluator. Do NOT use Python's eval(), exec(),
   ast.literal_eval, compile(), or any similar shortcut. If you use any of those the solution is REJECTED.
9. Error handling: raise ValueError (with a clear message) for ANY malformed input — e.g. "", "2+", "(1+2",
   "1 2", "*5", "2**", unbalanced parentheses, unexpected characters, an unknown identifier or function
   (e.g. "x", "foo(2)"). Raise ValueError on division by zero and modulo by zero too (convert Python's

10. Standard library only (you may import `math`). Include full docstrings and type hints.

OUTPUT: return ONLY the complete Python source code of calc.py. No markdown code fences, no prose
before or after the code. The first characters of your output must be valid Python.
